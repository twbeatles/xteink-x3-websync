# Xteink X3 WebSync Manager — 사용 설명서

뉴스·블로그·RSS 등을 모아 **e-ink 리더기(Xteink X3 / CrossPoint)** 에 최적화된 EPUB으로 만들고, Wi-Fi로 무선 전송하는 PC용 프로그램입니다.

|------|------|
| [README](../README.md) | 빠른 시작 |
| **이 문서** | 화면·설정·사이트 유형·문제 해결 |
| [개발자 가이드](DEVELOPER.md) | 코드 구조·테스트·빌드 |

---

## 1. 이 프로그램으로 할 수 있는 일

| 기능 | 설명 |
|------|------|
| 뉴스·블로그 수집 | 등록한 사이트의 새 글을 가져와 EPUB으로 만듦 |
| 무선 전송 | PC와 같은 Wi-Fi의 X3 기기로 바로 전송 |
| 중복 방지 | 이미 보낸 글은 다시 보내지 않음 (기기별 이력) |
| Calibre 연동 | PC 서재의 책을 골라 기기로 전송 |
| 예약 실행 | 매일 정해진 시간에 자동 동기화 |
| 선택 전송 | 프리뷰에서 원하는 기사만 골라 전송 |
| 기기 파일 관리 | File Transfer 모드에서 SD 목록·삭제·이동·업로드·정리 |

---

## 2. 준비물

1. **PC** — Windows 권장 (macOS/Linux에서도 소스 실행 가능)
2. **X3 기기** — CrossPoint 펌웨어, **PC와 같은 Wi-Fi** 에 연결
3. **기기 IP 또는 호스트** — 예: `192.168.0.25` 또는 `crosspoint.local`
4. (선택) **Calibre** — 서재 연동을 쓸 때만 설치

> **팁**: 기기 설정 화면이나 공유기 관리 페이지에서 IP를 확인할 수 있습니다.

---

## 3. 설치 및 실행

### 방법 A — EXE (Windows, 가장 간단)

1. `x3_websync.exe` 를 원하는 폴더에 둡니다.
2. 더블클릭으로 실행합니다.
3. 같은 폴더에 `config.json`, `output/`, `logs/`, `sync_history.db` 가 자동 생성됩니다.

### 방법 B — Python 소스

```bash
pip install -r requirements.txt
python x3_websync.py
```

### 백그라운드 동기화만 실행 (스케줄러용)

```bash
python x3_websync.py --sync
```

GUI 없이 설정된 사이트만 수집·전송합니다.

## 4. 화면 구성

프로그램은 위쪽 **탭 5개** + 아래 **동기화 버튼·로그** 로 나뉩니다.

| 탭 | 역할 |
|----|------|
| **뉴스 동기화** | 기기 IP, 사이트 목록, 스케줄, 로컬 파일 전송 |
| **Calibre 서재** | Calibre 도서 목록 조회·전송 |
| **동기화 이력** | 이미 보낸 글 확인·삭제(재전송 허용) |
| **기기 파일** | 기기 SD 파일 목록·삭제·이름 변경·이동·다운로드·업로드·오래된 EPUB 정리 |
| **고급 & 서버 설정** | EPUB 테마/합본, OPDS, 웹 대시보드, AI·번역 |

하단:

- **즉시 전체 뉴스 스크래핑 및 X3 동기화 실행** — 활성화된 사이트 전부 수집·전송
- **뉴스 프리뷰 (선택 동기화)** — 새 글 목록을 보고 일부만 전송
- **프로그램 상태 및 동기화 로그** — 진행 상황 확인

## 5. 처음 사용 — 5분 설정

### ① 기기 연결

1. **뉴스 동기화** 탭에서 **X3 주소**에 IP 또는 `crosspoint.local` 입력
2. **연결 확인** 클릭 → “연결 성공”이 뜨면 OK
3. (선택) 기기에서 **File Transfer** 모드로 진입한 뒤 **기기 파일** 탭에서 SD 목록을 관리할 수 있습니다
3. 기기가 여러 대면 **추가 X3 기기**에서 **기기 추가**

### ② 출력 폴더

- 기본값 `./output` 이면 프로그램 폴더 아래 `output`에 EPUB이 저장됩니다.
- **📂 열기**로 생성 파일을 바로 확인할 수 있습니다.

### ③ 사이트 등록

1. **사이트 추가** 클릭
2. 가능하면 **추천 프리셋**에서 골라 이름·유형·URL을 자동으로 채운 뒤 필요 시 수정  
   (또는 이름·유형·URL을 직접 입력)
3. 저장 후 목록에서 **활성 토글**로 on/off (비활성 사이트는 동기화에서 제외)

#### 사이트 유형 선택 가이드

| 유형 | 언제 쓰나요 | URL 예 |
|------|-------------|--------|
| `rss` | RSS/Atom 피드가 있을 때 (가장 안정적) | `https://example.com/feed` |
| `velog` | Velog 작가 (내부적으로 RSS) | `https://velog.io/@아이디` |
| `naver` | 네이버 블로그 | `https://blog.naver.com/아이디` |
| `naver_post` | ~~네이버 포스트~~ (**2025-04-30 서비스 종료** — 수집 불가) | — |
| `naver_cafe` | 네이버 **공개** 카페 (로그인·멤버 전용 글은 불가) | `https://cafe.naver.com/카페ID` |
| `tistory` | 티스토리 블로그 | `https://example.tistory.com` |
| `brunch` | 브런치 작가 프로필 | `https://brunch.co.kr/@작가ID` |
| `newneek` | 뉴닉 아티클 | `https://newneek.co/@newneek` |
| `soonsal` | 순살브리핑 뉴스레터 | `https://soonsal.com/newsletters/` |
| `moneyletter` | 어피티 머니레터 | `https://uppity.co.kr/newsletter/money-letter/` |
| `substack` | Substack 뉴스레터 | 서브스택 URL |
| `youtube` | 유튜브 채널 자막 → 텍스트 | 채널/영상 URL |
| `css` | 위 전용 유형이 없을 때 HTML 선택자로 수집 | 목록 페이지 URL |

> **추천**: 사이트 추가 대화상자의 **추천 프리셋**에서 Velog·뉴닉·한겨레·기술 블로그 등을 고르면  
> 이름·유형·URL이 자동으로 채워집니다.  
> `css`는 목록·제목·본문 **CSS 선택자**를 직접 넣어야 합니다.

#### 한국에서 쓰기 좋은 RSS 예시 (type=`rss` 또는 프리셋)

| 소스 | URL 예 |
|------|--------|
| Velog 작가 | `https://velog.io/@아이디` (`type=velog`) 또는 `https://v2.velog.io/rss/@아이디` |
| 한겨레 전체 | `https://www.hani.co.kr/rss/` (요약·이미지 위주일 수 있음) |
| 토스 / 카카오 / 우아한형제들 / 라인 기술 블로그 | 프리셋 목록 참고 |

언론·커뮤니티 **로그인 필수·페이월** 사이트는 지원하지 않습니다.

사이트별 옵션 예:

- **수집 개수(limit)** — 한 번에 가져올 글 수
- **이미지 포함** — e-ink에서는 보통 끔
- **번역** — 외국어 글일 때 사이트별로 지정
- **상세 페이지 수집** (`css`) — 목록만 있고 본문이 다른 페이지일 때

### ④ 첫 동기화

1. 하단 **즉시 전체 뉴스 스크래핑 및 X3 동기화 실행** 클릭
2. 로그에 수집·EPUB 생성·업로드 결과가 표시됨
3. 기기에서 새 EPUB이 보이는지 확인

이미 보낸 글은 이력에 남아 **다음부터는 새 글만** 보냅니다.

## 6. 자주 쓰는 기능

### 선택 동기화 (프리뷰)

1. **뉴스 프리뷰 (선택 동기화)** 클릭
2. 신규 기사 목록이 뜸
3. 원하는 글만 체크 후 전송

전체 자동 동기화 전에 “오늘 뭐가 쌓였는지” 볼 때 유용합니다.

### 매일 자동 동기화

1. **뉴스 동기화** 탭 → **자동 스케줄 설정**
2. 시·분 선택 후 **등록**
3. OS 스케줄러(Windows 작업 스케줄러 등)에 등록되어 매일 그 시간에 `--sync` 가 실행됩니다.

**해제**로 스케줄을 끌 수 있습니다.

### 로컬 파일 직접 전송

1. **로컬 파일 X3 직접 전송**에서 EPUB/PDF/MOBI/TXT 선택
2. **기기로 직접 전송**

뉴스 수집 없이 이미 있는 파일만 보낼 때 사용합니다.

### Calibre 서재

1. **Calibre 서재** 탭
2. `calibredb.exe` 경로 지정 (보통 Calibre 설치 경로)
3. **연결 확인 & 서재 로드**
4. 책을 선택(다중 선택 가능) 후 **즉시 전송**

### 동기화 이력 · 다시 보내기

1. **동기화 이력** 탭에서 전송 기록 확인
2. 특정 글을 다시 보내려면 해당 항목 **삭제** 후 다시 동기화
3. **전체 이력 초기화**는 모든 중복 방지 기록이 지워지므로 신중히 사용

### 사이트 설정 백업 (수동)

- **설정 내보내기** — 사이트 목록을 JSON으로 저장 (다른 사람과 공유·일회성 백업)
- **설정 가져오기** — 다른 PC/재설치 후 목록 복원 (URL 중복은 건너뜀)

### 클라우드 백업 동기화 (OneDrive 등)

여러 PC·재설치 환경에서 **사이트 구독 목록**과 **이미 보낸 글 목록(전송 이력)** 을 자동으로 맞추고 싶을 때 사용합니다.

**고급 & 서버 설정** 탭 → **클라우드 백업 동기화 (OneDrive 등)**:

1. OneDrive/Dropbox 등 동기화되는 폴더를 하나 만듭니다 (예: `OneDrive\XteinkWebSync`)
2. 해당 폴더를 **동기화 폴더**로 지정하고 **백업 동기화 사용**을 켭니다
3. **지금 동기화**를 한 번 실행하거나, 앱 시작·뉴스 동기화 시 자동으로 반영됩니다

폴더에 생성되는 파일:

| 파일 | 내용 |
|------|------|
| `sites.json` | 사이트 구독 설정 (포터블) |
| `synced_posts.json` | 전송 완료 글 목록 (중복 방지용 이력) |
| `manifest.json` | 스키마 버전·내보내기 시각 |

**포함되는 것**: 사이트 목록, 전송 이력(URL·기기 IP 키·제목 등)  
**포함되지 않는 것**: 기기 주소 목록, Calibre 경로, `output_dir`, API 키/토큰, 서버 포트 등 PC 로컬 값

**병합 방식**:

- 이력: 여러 PC 이력을 **합집합**으로 합칩니다 → 한 PC에서 이미 보낸 글을 다른 PC가 다시 보내지 않습니다
- 사이트: 동일 URL은 더 최근 내보내기 쪽이 우선하고, 각 PC에만 있는 사이트는 유지합니다

**이력 탭**의 **이력 JSON 내보내기/가져오기**로 클라우드 폴더 없이도 일회성 백업·복원이 가능합니다.

주의:

- `sync_history.db` SQLite 파일을 OneDrive에 직접 두지 마세요 (잠금·손상 위험). JSON 미러를 사용하세요
- OneDrive가 아직 내려받는 중이면 다음 동기화 때 다시 시도됩니다
- 기기 IP·폰트 등 로컬 설정은 PC마다 따로 지정하세요

## 7. 고급 설정 (선택)

**고급 & 서버 설정** 탭에서 다룹니다.

### EPUB 스타일

| 항목 | 설명 |
|------|------|
| **사이트별 개별 EPUB** | 사이트마다 파일 하나 (기본) |
| **일간 합본 EPUB** | 그날 모은 글을 한 권으로 |
| **테마** | `default`, `serif_classic`, `sans_modern`, `dark_eink` 등 |
| **커스텀 CSS** | 직접 만든 CSS 파일 경로 |

뉴스 동기화 탭의 **폰트 / 글자 크기 / 줄 간격 / 표지 생성**도 EPUB 읽기 편의에 영향을 줍니다.

### OPDS 서버

- 생성한 EPUB을 OPDS 카탈로그로 제공
- 기기 OPDS 클라이언트로 목록을 보고 받을 수 있음
- 기본은 **이 PC(localhost)만** 접근
- **LAN 공개** 시 API 키 인증이 필요합니다 (헤더 `Authorization: Bearer …` 또는 `X-Api-Key` 권장)  
- URL 쿼리 `?api_key=` 는 로그 유출 위험으로 **기본 비활성**입니다. 구 클라이언트만 `X3_OPDS_ALLOW_QUERY_API_KEY=1` 환경변수로 허용

### 웹 대시보드

- 브라우저에서 동기화 실행·로그 확인
- API 토큰으로 로그인 (토큰은 `config.json`에 저장)
- LAN 공개는 **신뢰할 수 있는 네트워크**에서만 사용 (HTTP 평문)

### Calibre 폴더 감시

- 지정 폴더에 새 파일이 생기면 자동으로 기기에 전송
- (소스 실행 시) `watchdog` 패키지가 필요할 수 있음

### AI 요약 / 번역

- **AI 요약**: OpenAI 또는 로컬 Ollama — 본문이 외부 API로 갈 수 있음
- **번역**: googletrans 또는 LibreTranslate  
  - googletrans: 사이트별 번역 설정만으로도 동작하는 경우가 많음  
  - libretranslate: 전역 활성화 필요

선택 기능(표지 이미지, YouTube 자막, 폴더 감시, googletrans 등)은 EXE 빌드에 포함되지 않을 수 있습니다. 소스 실행 시:

```bash
pip install -r requirements-optional.txt
```

## 8. 데이터가 저장되는 위치

프로그램(또는 exe)이 있는 폴더 기준:

| 경로 | 내용 |
|------|------|
| `config.json` | 기기·사이트·스케줄 등 모든 설정 |
| `sync_history.db` | 전송 이력 (중복 방지) |
| `output/` | 생성된 EPUB |
| `logs/sync_YYYY-MM-DD.log` | 날짜별 실행 로그 |

설정을 백업할 때는 최소한 **`config.json`** 과 필요 시 **`sync_history.db`** 를 복사하세요.  
여러 PC 동기화는 위 **클라우드 백업 동기화** 폴더(`sites.json` + `synced_posts.json`)를 권장합니다.

## 9. 문제 해결

| 증상 | 확인 사항 |
|------|-----------|
| 연결 확인 실패 | PC·기기가 **같은 Wi-Fi**인지, IP가 맞는지, 기기 전원이 켜져 있는지 |
| 동기화해도 새 글 없음 | 사이트가 **활성화**인지, 이미 이력에 있는 글인지, 수집 limit·URL이 맞는지 |
| 특정 글만 다시 보내고 싶음 | **동기화 이력**에서 해당 항목 삭제 후 다시 동기화 |
| 스케줄이 안 돌아감 | 스케줄 **등록** 상태 확인, Windows는 작업 스케줄러 권한·경로 확인, `logs/` 로그 확인 |
| “동기화가 이미 실행 중” | 다른 창/스케줄러 동기화가 끝날 때까지 대기 (정상 동작) |
| 기기에서 파일이 안 보이거나 오류 | 한글·특수문자 파일명은 전송 시 정리됨. 최신 버전 사용 여부 확인 |
| EXE에서 YouTube/표지/감시 안 됨 | 해당 기능은 선택 의존성이라 EXE에 없을 수 있음 → 소스 실행 + optional 패키지 |

로그 확인:

- 화면 하단 로그창
- **고급 & 서버 설정** → **로그 폴더 열기**

## 10. 권장 일상 루틴

1. **한 번만**: 기기 IP 등록 → 자주 읽는 사이트 등록 → 스케줄 등록 (예: 매일 07:00)
2. **매일 아침**: 자동으로 EPUB이 기기로 전송됨
3. **가끔**: 프리뷰로 골라 보내기, Calibre에서 책 전송, 이력 정리

## 11. 보안 참고 (짧게)
